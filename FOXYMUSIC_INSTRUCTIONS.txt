FoxyMusic instructions
======================

Project overview (maintainers): see also docs/README.md in the repository root for
architecture, layout, and a short summary of what FoxyMusic is.


What this app is
----------------
FoxyMusic is an Android music app with:

- Flutter frontend UI at the repository root:
  - pubspec.yaml
  - lib/main.dart
  - .android/
  - .ios/

- Kotlin/Android backend and host app:
  - app/src/main/java/com/foxymusic/
  - app/build.gradle
  - settings.gradle

Important: the old foxy_flutter_ui folder is legacy. New Flutter UI work should be done in:

  D:\FoxyMusic\lib\main.dart

Do not add new UI work under:

  D:\FoxyMusic\foxy_flutter_ui

How the app works
-----------------
The Android app hosts the Flutter UI. Flutter handles the visible screens, player sheet,
library, settings sheet, lyrics UI, search UI, and home layout.

Kotlin handles the native app logic:

- Playback and queue:
  app/src/main/java/com/foxymusic/MusicPlayer.kt
  app/src/main/java/com/foxymusic/FoxyPlayerConnection.kt

- Flutter method/event channel bridge:
  app/src/main/java/com/foxymusic/FoxyFlutterBridge.kt
  app/src/main/java/com/foxymusic/FoxyFlutterChannels.kt

- YouTube Music search/feed/lyrics/account helpers:
  app/src/main/java/com/foxymusic/YTMusicApi.kt
  app/src/main/java/com/foxymusic/LyricsRepository.kt
  app/src/main/java/com/foxymusic/FoxyAccount.kt

- Library, likes, downloads, history:
  app/src/main/java/com/foxymusic/FoxyLibraryStore.kt
  app/src/main/java/com/foxymusic/FoxyDownloadManager.kt

- Settings persistence and theme:
  app/src/main/java/com/foxymusic/FoxySettings.kt
  app/src/main/java/com/foxymusic/FoxyDynamicTheme.kt

- Foreground media session / system notification:
  app/src/main/java/com/foxymusic/FoxyMediaSessionService.kt
  (MainActivity requests POST_NOTIFICATIONS on Android 13+.)


Flutter <-> Kotlin channels
---------------------------
Flutter talks to Kotlin through these channels in lib/main.dart:

  MethodChannel: foxy_music/methods
  EventChannel : foxy_music/events

Common Flutter -> Kotlin method calls:

- getAppearance
- setAppearance
- homeFeed
- moodMix
- search
- libraryFeed
- accountInfo
- playQueue
- togglePlayPause
- previous
- next
- seekTo
- toggleShuffle
- cycleRepeatMode
- like
- download
- addToQueue
- enqueuePlayNext
- removeFromQueue
- sleepTimer
- cancelSleepTimer
- lyrics
- openSystemEqualizer
- openWebLogin
- openExternalUrl

Kotlin sends player state updates to Flutter through the event channel with:

  type = playerState

Each playerState map (under key "state") includes at least:

  - isPlaying, isBuffering, positionMs, durationMs, queueIndex
  - shuffleEnabled, repeatMode, error
  - queue (list of song maps), currentSong (song map or null)
  - dynamicSongColors (bool, mirrors settings)
  - songAccentArgb (nullable number, ARGB when dynamicSongColors is true and artwork accent
    has been computed; null otherwise)

Kotlin sends sleep timer updates with:

  type = sleepTimer

The Flutter UI listens for playerState and updates the mini player, now playing sheet,
queue, lyrics position, buffering state, and playback controls. Do not add a second
independent Dart-only artwork palette unless you also update Kotlin; keep one pipeline.


Offline downloads and metadata
-------------------------------
Progressive downloads (FoxyDownloadManager) write media files under the app external files
downloads folder. When a download completes, FoxyLibraryStore writes:

  <videoId>.foxy-meta.json

next to the media file with title, artist, thumbnail, artworkUrl, duration, album.

On boot, loadDownloadedSongs reads only known media extensions and skips *.foxy-meta.json.
If meta is missing (older installs), songs still get YouTube thumbnail URLs from the
stored video id.


Editing guide
-------------
For frontend UI changes:

  D:\FoxyMusic\lib\main.dart

For backend/player/channel changes:

  D:\FoxyMusic\app\src\main\java\com\foxymusic

If you add a new Flutter button that needs native behavior:

1. Add a method name in:
   app/src/main/java/com/foxymusic/FoxyFlutterChannels.kt

2. Handle the method in:
   app/src/main/java/com/foxymusic/FoxyFlutterBridge.kt

3. Call it from Flutter using:
   _method.invokeMethod('methodName', optionalMap)

If you add new fields to playerState for Flutter, update:

  - FoxyFlutterBridge.kt (emit payload)
  - lib/main.dart (parsing / UI as needed)

Gradle tips (this machine)
--------------------------
If the daemon fails to bind or file-lock errors appear during shutdown, you can force a
dedicated Gradle user home for one build:

  set GRADLE_USER_HOME=D:\FoxyMusic\.gradle-user-home4
  .\gradlew.bat assembleRelease --no-daemon --console=plain

On this machine Gradle can finish successfully but still print or return an error during
daemon shutdown:

  IllegalStateException: Cannot start managing file contention because this handler has been closed.

If this happens, check the daemon log and APK timestamps before assuming the build failed.

The useful success line looks like:

  BUILD SUCCESSFUL

The real output folder to check is:

  D:\FoxyMusic\app\build\outputs\apk\release   (release APK splits)
  D:\FoxyMusic\app\build\outputs\bundle\release   (release .aab when you run bundleRelease)
  D:\FoxyMusic\app\build\outputs\apk\debug     (debug)

Make sure the LastWriteTime of the APK files changed after the build.

Build release APKs
------------------

  .\gradlew.bat assembleRelease

Force a fresh release rebuild:

  .\gradlew.bat assembleRelease --rerun-tasks --no-daemon --console=plain --stacktrace

Release APKs are created under:

  D:\FoxyMusic\app\build\outputs\apk\release

Expected split APK names:

  app-arm64-v8a-release.apk
  app-armeabi-v7a-release.apk

The app currently uses debug signing for release builds in app/build.gradle:

  signingConfig signingConfigs.debug

For Play Store or public production release, replace this with a real release keystore.

Media notification checklist
------------------------------
- Channel id used by the app for playback: foxy_playback (see FoxyMediaSessionService).
- User must allow notifications on Android 13+ (MainActivity requests POST_NOTIFICATIONS).
- If the notification does not appear while audio plays, check OEM battery restrictions and
  notification permission for FoxyMusic.
