FoxyMusic — maintainer & agent guide
=====================================

Repository: hybrid Android (Kotlin) + Flutter UI at repo root.
GitHub: https://github.com/sparkn2008-del/FoxyMusic

BUILD
-----
From repo root (Windows):

  .\gradlew.bat assembleDebug
  .\gradlew.bat assembleRelease
  .\gradlew.bat bundleRelease

Outputs:
  app\build\outputs\apk\debug\     (per-ABI splits)
  app\build\outputs\apk\release\
  app\build\outputs\bundle\release\app-release.aab

Release signing: configure key.properties (see app/build.gradle). Until then release may use debug signing.

Flutter analyze/format: use your Flutter SDK (Puro or PATH). Dart UI lives in lib/main.dart.

VERSION
-------
Bump in app/build.gradle:
  versionCode (int, monotonic)
  versionName (string, shown in UI and compared to GitHub tag_name)

Current example: versionCode 5, versionName "1.2.1"

FLUTTER ↔ KOTLIN BRIDGE
-----------------------
Channels (FoxyFlutterChannels.kt, FoxyFlutterBridge.kt):

  MethodChannel: foxy_music/methods
  EventChannel:  foxy_music/events

Dart:
  const _method = MethodChannel('foxy_music/methods');
  const _events = EventChannel('foxy_music/events');

Key methods (add name in FoxyFlutterChannels.Methods, handle in FoxyFlutterBridge, call from Dart):
  getAppearance / setAppearance  — theme, playback prefs, updater toggles
  getPlayerState                 — queue, transport, crossfade, lyrics pref
  checkGitHubRelease             — manual update check (returns map)
  getAppVersion                  — { versionName, versionCode }
  openExternalUrl                — browser / APK download link

Key events (type field on event map):
  playerState
  appearanceChanged
  libraryFeedChanged
  updateAvailable                — { update: { ok, updateAvailable, tagName, htmlUrl, downloadUrl, body, ... } }

APP UPDATER (SimpMusic-style)
-----------------------------
Kotlin:
  FoxyGithubUpdate.kt   — GET releases/latest, semver-ish compare, prefer arm64 APK asset
  FoxyUpdatePrefs.kt    — 24h throttle (last_check_ms), last_notified_tag
  FoxyUpdateNotifier.kt — notification channel "foxy_app_updates" (ID foxy_app_updates)

Settings (FoxyCustomization / FoxySettings):
  autoCheckUpdates      — default true; background check when EventChannel connects
  updateNotifications   — default true; system notification once per new tag

Flow:
  1. Flutter subscribes to EventChannel → bridge may run throttled auto-check.
  2. If newer tag than installed versionName → emit updateAvailable + optional notification.
  3. Settings → "Check for updates now" calls checkGitHubRelease (always runs, no throttle).
  4. User opens release page or APK via openExternalUrl.

Manual check does not show notification unless you wire it; auto-check uses deliverUpdateCheckResult(notifyUser=true).

GLASS UI (Flutter)
------------------
  _FoxyGlassTint      — scroll surfaces (no BackdropFilter; avoids blur reload on scroll)
  _FoxyGlassButton    — blur: true only on fixed chrome (bottom nav, icon buttons)
  Player screen       — excluded from glass refactors unless requested

NATIVE LAYOUT
-------------
  app/src/main/java/com/foxymusic/  — player, library, bridge, updater
  lib/main.dart                     — primary Flutter UI
  foxy_flutter_ui/                  — legacy; do not add new features there
