QUICK ORIENTATION
-----------------
  Kotlin owns truth for playback, persistence, networking, recognition, updates.
  Flutter owns screens, animations, and most user-visible layout.
  Never add new UI under foxy_flutter_ui/ — it is legacy and gitignored.

  Primary Dart entry : lib/main.dart
  Bridge             : FoxyFlutterChannels.kt + FoxyFlutterBridge.kt
  Player engine      : MusicPlayer.kt + FoxyMediaSessionService.kt
  YT API             : YTMusicApi.kt, StreamExtractor.kt


BUILD COMMANDS (Windows, repo root)
-----------------------------------
  .\gradlew.bat assembleDebug
  .\gradlew.bat assembleRelease
  .\gradlew.bat bundleRelease

Outputs
  app\build\outputs\apk\debug\
  app\build\outputs\apk\release\     (per-ABI splits)
  app\build\outputs\bundle\release\app-release.aab

Requirements
  JDK 17+
  Android SDK API 34+
  Flutter SDK (for lib/ — Puro path example:
    C:\Users\<user>\.puro\envs\stable\flutter\bin\flutter.bat)

Release signing
  Copy key.properties.example → key.properties
  Fill storeFile, storePassword, keyAlias, keyPassword
  Without key.properties, release may sign with debug key (see app/build.gradle)

Useful checks

  .\gradlew.bat :app:compileDebugKotlin

FLUTTER ↔ KOTLIN BRIDGE
-----------------------
Channels (FoxyFlutterChannels.kt)

  MethodChannel : foxy_music/methods
  EventChannel  : foxy_music/events

Dart (lib/main.dart)

  const _method = MethodChannel('foxy_music/methods');
  const _events = EventChannel('foxy_music/events');

When adding a method
  1. Add constant in FoxyFlutterChannels.Methods
  2. Handle in FoxyFlutterBridge.onMethodCall
  3. Call from Dart via _method.invokeMethod(...)
  4. Document here + docs/README.md if public-facing

Core methods (non-exhaustive)
  getAppearance / setAppearance   Theme, playback prefs, updater toggles
  getPlayerState                  Queue, transport, crossfade, lyrics pref
  playQueue / pause / resume / seekTo / next / previous
  download / removeDownload       Offline library
  getLibraryFeed / getHomeFeed    Shelves + library snapshots
  checkGitHubRelease              Manual update check → map
  getAppVersion                   { versionName, versionCode }
  openExternalUrl                 Browser / APK download
  startRecognition / stopRecognition / getRecognitionHistory
  backupLibrary / restoreLibrary  FoxyBackup.kt

Core events (map with "type" field)
  playerState           Full player snapshot (emit on transport changes)
  appearanceChanged     Settings/theme applied
  libraryFeedChanged    Library or downloads mutated
  updateAvailable       { update: { ok, updateAvailable, tagName, ... } }
  recognitionResult     Mic identification finished
  recognitionHistoryChanged

Player state hygiene
  FoxyFlutterBridge merges live ExoPlayer position/duration into snapshots.
  Buffering is suppressed when effectively playing (avoids stuck mini-player).


HOME FEED
---------
Kotlin assembly : FoxyFlutterBridge.kt (home feed builder)
Fallback rows   : YTMusicApi.fallbackRecommendationSections()
Dart layout     : _homeSectionLayout() in lib/main.dart
Dart suppression: _isSuppressedHomeSection()

Layout kinds (_HomeSectionLayout)
  square  — horizontal card rail (resume, replayed, downloaded)
  grid    — 2-column shelf (today on foxy, india pulse, hidden gems, covers…)
  radio   — compact two-row radio starters
  mixes   — mix cards
  video   — wide video thumbnails
  chart   — chart shelf
  artist  — circular artist avatars
  cards   — vertical song list

Explore launcher (Moods / Genres / Charts) inserts after "New releases"
via _homeFeedSlivers() in _HomeTabState.

Spacing constants (lib/main.dart)
  _kHomeShelfGap        Vertical gap between shelves (14)
  _kHomeShelfTitleGap   Title → content (8)
  _kHomeExploreRailGap  Between explore rails (14)
  _HomeShelfShell       Wrapper applying shelf top gap

UPDATER (GitHub releases/latest)
--------------------------------
Kotlin
  FoxyGithubUpdate.kt    Semver-ish compare, prefers arm64-v8a APK asset
  FoxyUpdatePrefs.kt     24h throttle (last_check_ms), last_notified_tag
  FoxyUpdateNotifier.kt  Channel "foxy_app_updates"

Settings keys (via appearance map)
  autoCheckUpdates      default true
  updateNotifications   default true

Flow
  1. Flutter subscribes to EventChannel → bridge may run throttled auto-check
  2. Newer tag than installed versionName → updateAvailable event + optional notification
  3. Settings "Check for updates now" → checkGitHubRelease (no throttle)
  4. openExternalUrl for release page or APK

AGENT WORKFLOW
--------------
1. Read this file + grep the feature area before editing.
2. Kotlin first if playback, downloads, API, or persistence changes.
3. Expose bridge contract, then wire Flutter.
4. Keep diffs focused — match existing naming and patterns.
5. Run flutter analyze and/or gradlew compile when possible.
6. Update changelog.txt for user-visible changes.
7. Do not commit unless the user explicitly asks.
