FoxyMusic - Maintainer & Agent Blueprint
========================================

Repository : https://github.com/sparkn2008-del/FoxyMusic
Stack      : Android Kotlin + Media3 core, Flutter UI in repo root
Package    : com.foxymusic
Current    : versionName 1.3.0, versionCode 7

This file is the working blueprint for agents, maintainers, and future UI work.
The public README is README.md. Contributor architecture notes also live in
docs/README.md.


CORE RULE
---------
Kotlin owns playback truth, persistence, networking, stream extraction,
recognition, downloads, update checks, and native file pickers.

Flutter owns screens, layout, user interactions, animations, bottom nav,
mini-player, settings UI, and the visual Home/Search/Library/Player surfaces.

Do not add new UI under foxy_flutter_ui/. It is legacy and ignored. Use lib/.

BRIDGE CONTRACT
---------------
Channels:

  MethodChannel : foxy_music/methods
  EventChannel  : foxy_music/events

Dart constants:

  const _method = MethodChannel('foxy_music/methods');
  const _events = EventChannel('foxy_music/events');

When adding a bridge method:

  1. Add the constant in FoxyFlutterChannels.Methods.
  2. Implement it in FoxyFlutterBridge.onMethodCall.
  3. Call it from lib/main.dart with _method.invokeMethod(...).
  4. Update this file if the behavior is user-visible or architectural.

Important methods:

  getAppearance / setAppearance
    Theme, playback prefs, update prefs, Home background flags.

  getPlayerState
    Current song, queue, index, position, duration, playback flags, settings.

  playQueue / pause / resume / togglePlayPause / seekTo / next / previous
    Player transport.

  homeFeed
    Hybrid Home feed sections used by Flutter shelves.

  searchAll
    Foxy categorized search payload:
    songs, videos, albums, artists.

  resolveArtistProfile
    Artist page payload: best artist profile, songs, albums.

  songMenuContext
    Lightweight menu context for overflow sheets. Use this instead of loading
    the full library feed inside every three-dot menu.

  libraryFeed
    Liked, history, downloads, most played, playlists, recognized tracks.

  download / removeDownload
    Offline library mutations.

  pickHomeBackground / clearHomeBackground
    Native image picker and custom Home background storage.

  startRecognition / stopRecognition / getRecognitionHistory
    Music recognition flow.

  checkGitHubRelease / getAppVersion / openExternalUrl
    Updater and external links.

Important events:

  playerState
    Native player snapshot. Flutter must avoid rebuilding the whole Home page
    from progress-only changes.

  appearanceChanged
    Settings/theme/background changes.

  libraryFeedChanged / libraryDownloadsChanged / libraryDownloadProgress
    Library and download updates.

  updateAvailable
    GitHub updater result.

  recognitionResult / recognitionHistoryChanged
    Song recognition updates.


HOME SCREEN BLUEPRINT
---------------------
Home UI lives in _HomeTab and _HomeTabState in lib/main.dart.

Current design goals:

  - AMOLED black background.
  - Premium, less crowded, easy to scroll.
  - No huge repeated shelves stacked endlessly.
  - Mini-player and bottom nav must not cover cards.
  - First render should feel fast using cached content and skeleton shells.
  - Home shelf artwork uses thumbnail/homeArtwork routes for efficiency.
  - Player artwork may still use high quality artwork.

Initial load behavior:

  - _HomeCache keeps last Home feed sections in memory.
  - If Home is refreshing and cached sections exist, keep content visible.
  - If no sections exist yet, show _HomeLoading skeleton shells.
  - Home feed native jobs have timeout guards in FoxyFlutterBridge so optional
    shelves do not block the whole Home page.

Top structure:

  1. Foxy Pick hero
     _HomeSpotlight
     Large rounded rectangle, currently compacted from the original oversized
     version. Uses clear artwork, not heavy blur. Contains title, artist, Play,
     and Radio.

  2. Quick chips
     _HomeQuickActionChips
     Moods, Genres, Charts, Downloads, Radio, History.

     Moods/Genres/Charts route into Library category drill-ins:
       _LibraryTabState.scopeMoods
       _LibraryTabState.scopeGenres
       _LibraryTabState.scopeCharts

     Downloads routes to Library.
     History routes to Search with query/history intent.

  3. Feed shelves
     _HomeFeedSection picks a visual layout from _homeSectionLayout().

Layout routing:

  _HomeSectionLayout.square
    Horizontal square cards. Used for resume/replay/downloaded/fresh rows.
    Cards are true square cards with full artwork and bottom gradient text.

  _HomeSectionLayout.chart
    Wide horizontal banner row. Used for Charting now and Trending now.
    Do not add ranking badges here. No square cards for these sections.

  _HomeSectionLayout.artist
    Transparent artist rail. Circular artwork only, no glass/card background.
    Artist name comes from song.title; subtitle/subscribers stay secondary data.

  _HomeSectionLayout.radio
    Compact two-row station tiles.

  _HomeSectionLayout.mixes
    Compact listen-together/mix cards. These must not become oversized.

  _HomeSectionLayout.video
    Compact video-style banner cards with full artwork and overlay text.

  _HomeSectionLayout.cards
    Vertical/row song card list.

Home card rules:

  - Normal shelf cards should stay about 15 percent shorter than the old tall
    layout.
  - Square cards should be square.
  - Charting now and Trending now are exceptions: keep them wide banner cards.
  - Artwork should cover the full card.
  - Text must be inside the border and must use ellipsis/maxLines.
  - Use bottom gradient or blur-gradient overlays to keep titles readable.
  - Horizontal rows must start from the first card, not centered.
  - Avoid huge empty gaps between sections.

Bottom spacing:

  _HomeTab receives bottomContentPadding from the shell.
  This padding must include mini-player height + nav bar height + safe area.
  Cards should never hide behind mini-player/nav.

Mini-player:

  Floating glass mini-player sits above bottom nav.
  Avoid full Home rebuilds on progress-only player updates.


HOME FEED SOURCE BLUEPRINT
--------------------------
Native feed builder:

  FoxyFlutterBridge.kt -> HOME_FEED

Private research folders:

  SimpMusic-main/
    Root-level local reference for categorized discovery behavior.

  ArchiveTune-main/
    Root-level local reference for broader shelf/category behavior.

Implementation rule:

  Use those folders only to understand behavior and edge cases. FoxyMusic code,
  class names, comments, labels, and docs must describe Foxy-owned concepts.
  Do not leave source comments or UI labels that imply external ownership.

Current feed shape:

  - Hybrid personalized shelves from history/library.
  - Search/recommendation fallback rows from YTMusicApi.
  - Featured artists section from YTMusicApi.searchArtistProfiles.
  - Downloaded but forgotten appears only when downloaded songs exist.
  - Repetitive shelves such as Most replayed / Today on Foxy / High quality
    picks should be merged conceptually into Resume/Foxy Picks where possible.

Performance rules:

  - Load only the first visible sections initially.
  - Increase visible section count in batches while scrolling.
  - Avoid more than 6 items in grid-like sections.
  - Cache artwork thumbnails.
  - Use homeArtwork / thumbnail routes for shelves.
  - Keep player high-quality artwork path separate.
  - Use withTimeoutOrNull around slow optional native feed jobs.


SEARCH BLUEPRINT
----------------
Search UI lives in:

  _SearchTab
  _SearchResultsPage
  _FoxySearchResultRow
  _ArtistPage

Native categorized search:

  FoxyFlutterBridge.kt -> SEARCH_ALL
  YTMusicApi.searchAll(query, limit)

Search categories:

  All
  Songs
  Videos
  Albums
  Artists

How it works:

  - SEARCH_ALL returns songs/videos/albums/artists.
  - _SearchTab caches recent query snapshots in _SearchTabState._resultCache.
  - Albums and Artists are shown in the inline All grouping, not hidden behind
    only song rows.
  - Albums tab shows album rows.
  - Artists tab shows artist rows.
  - Artist rows show artist name from song.title and open _ArtistPage.
  - Song/video/album rows may use the overflow menu.
  - Artist rows should not open the heavy song overflow menu.

Artist page:

  _ArtistPage calls resolveArtistProfile.
  It shows top artist songs and albums.
  Artist artwork can be passed from search/home or resolved natively.


LIBRARY BLUEPRINT
-----------------
Library UI lives in _LibraryTab and _LibraryTabState.

Collection scopes:

  _scopeHub
  _scopeLiked
  _scopeHistory
  _scopeDownloads
  _scopeMostPlayed
  _scopePlaylists
  _scopeRecognized

Browse/category scopes:

  scopeMoods
  scopeGenres
  scopeCharts
  scopeArtists

How it works:

  - The Library hub shows "Your collection" tiles and "Browse music" tiles.
  - Moods, Genres, Charts, and Artists open category drill-ins.
  - Category drill-ins do not depend on libraryFeed loading.
  - Category tiles call widget.onDiscoverSearch(query), which routes into the
    Search tab with a prefilled query.
  - Home quick chips route into these same Library scopes.

Current category examples:

  Moods
    Relax, Sleep, Focus, Drive, Party, Phonk.

  Genres
    Bollywood, Punjabi, Indie, Hip hop, EDM, Lofi.

  Charts
    India Top, Global, Trending, New Hot.

  Artists
    Top artists, India artists, Pop artists, Indie artists.

Downloads:

  - Downloads scope shows offline songs and active download progress.
  - Downloaded but forgotten Home shelf should only exist when downloads exist.

Playlists:

  - Local/user playlists are stored through FoxyUserPlaylists.
  - YouTube playlists may appear when account integration is available.


THREE-DOT MENU BLUEPRINT
------------------------
Main function:

  showFoxySongOverflowMenu(...)

Rules:

  - Use songMenuContext for quick state/context.
  - Do not call libraryFeed inside every menu open; it causes lag.
  - Menu contents should be contextual:
      song rows: play next, add queue, like, download, playlist, artist/search
      album rows: album/search actions, avoid song-only assumptions
      artist rows: usually navigate/open artist, no heavy song menu
      queue rows: include remove from queue when requested
      downloads: include remove download where appropriate
  - Keep bottom sheet lightweight and avoid expensive work before first paint.


SETTINGS BLUEPRINT
------------------
Settings UI is in lib/main.dart around _SettingsSheet and related sections.
Persistence is native in FoxySettings.kt via getAppearance/setAppearance.

Important working areas:

  Playback
    Audio quality tier, crossfade, normalize, skip silence, replay gain.

  Downloads/cache
    Download quality, cache cleanup, lyrics cache cleanup, library repair.

  Lyrics
    Lyrics source preferences, romanization/transliteration behavior.

  Recognition
    Recognition settings and history controls.

  Appearance
    AMOLED/glass treatment, custom Home background.

  Updates
    Manual update check, auto-check, notifications.

Removed/merged preference:

  Network preference is intentionally removed from UI. Audio quality preference
  is the single user-facing control for stream quality/network use.

Custom Home background:

  - Flutter calls pickHomeBackground.
  - MainActivity stores the image through FoxyHomeBackground.
  - MainActivity sets homeBackgroundEnabled = true after successful pick.
  - clearHomeBackground removes it and sets homeBackgroundEnabled = false.
  - Flutter _loadAppearance only applies homeBackgroundPath when enabled.


AUDIO QUALITY BLUEPRINT
-----------------------
Native implementation:

  StreamExtractor.kt
    qualityPreferenceForTier()
    qualityPreferenceScore()

Current tier intent:

  Low
    Pick the least practical stream. Target very low bitrate.

  Normal
    Target around 220 kbps.

  High
    Target around 320 kbps.

  Ultra
    Prefer streams above 320 kbps when available, especially Opus, FLAC, AAC.
    If no 320+ stream exists, pick the best available stream.

Important:

  - The player should keep high-quality playback logic.
  - Home shelf thumbnails should stay efficient and must not force player-grade
    artwork or stream quality.


ARTWORK BLUEPRINT
-----------------
Song model exposes multiple artwork concepts in Flutter:

  artwork
    Default artwork.

  homeArtwork
    Efficient Home/shelf thumbnail route. Use this in Home shelves and small
    rows where possible.

  highQualityArtwork
    Player, artist page, large artwork, and high-detail surfaces.

Rules:

  - Home cards should generally use homeArtwork.
  - Player/Now Playing can use highQualityArtwork.
  - Cards should set fit/cover behavior through _Artwork usage and full-card
    Stack layouts.
  - Use gradient overlays for text readability instead of putting text outside
    the card border.

Main sheet:

Mini-player:

UPDATER BLUEPRINT
-----------------
Native files:

  FoxyGithubUpdate.kt
  FoxyUpdatePrefs.kt
  FoxyUpdateNotifier.kt

Flow:

  - Compares installed versionName to GitHub releases/latest.
  - Auto-check is throttled.
  - Manual check from settings bypasses throttle.
  - updateAvailable event can show UI and notification.
  - openExternalUrl opens release page or APK asset.


DESIGN RULES
------------
Home:

  - AMOLED black first.
  - Rounded cards around 18-24 px, unless compact rows need less.
  - Thin white/gray borders.
  - White bold section titles.
  - Gray secondary text.
  - No large random gaps.
  - Section margin should stay consistent.
  - Chips belong near the top, directly after the hero.
  - Avoid repetitive shelf names and duplicate content.

Cards:

  - Artwork covers the whole card.
  - Text stays inside the card.
  - Use maxLines and TextOverflow.ellipsis.
  - Use bottom gradient/blur-gradient for readability.
  - Horizontal rows start at first item.

Artists:

  - Artist Home shelf has no card background.
  - Use circular artwork.
  - Show artist name clearly.

Charts/Trending:

  - Use wide horizontally scrollable banner cards.
  - Do not square them.
  - Do not show numbering badges.

Glass:

  - _FoxyGlassTint: cheap scroll surfaces, avoid excessive BackdropFilter.
  - _FoxyGlassButton blur: true only for fixed chrome and small controls.
  - Avoid blur-heavy repeated list items.

AGENT WORKFLOW
--------------
Before edits:

  1. Read this file.
  2. Use rg to find the current feature area.
  3. Check whether the behavior belongs in Kotlin or Flutter.

Implementation:

  - Kotlin first for playback, downloads, API, recognition, settings storage.
  - Flutter first for layout, visual behavior, tabs, menus, interactions.
  - Keep bridge contracts explicit.
  - Match existing naming and patterns.
  - Keep diffs focused.
  - Do not revert user changes.

Validation:

  - Run dart format for edited Dart files.
  - Run dart analyze lib/main.dart for main UI changes.
  - Run .\gradlew.bat :app:compileDebugKotlin when bridge/native/app compile
    confidence is needed.
  - Note known project-level analyze issues separately.

Documentation:

  - Update this file for architectural changes.
  - Update changelog.txt for user-visible release changes.
  - Do not commit unless the user explicitly asks.
